# Features (functional requirements)

One **bundle** per feature (board, inbox, comments, detail panel, ...). Each feature folder
carries `spec.md`, `plan.md`, `testing.md`, and `ui.md` (if it has a UI). See
[../README.md](../README.md) for the bundle convention.

No features have been carved into bundles yet: [`../scope.md`](../scope.md) currently holds the
holistic functional spec for the whole product. Split a per-feature `spec.md` out of it as each
feature is specced in depth.
